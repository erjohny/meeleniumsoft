# meeleniumsoft — Конструктор сайтов (React)

React-версия проекта **buildix** — конструктор сайтов «как мини-Tilda»: собираешь сайт из блоков
без кода, правишь прямо на холсте, публикуешь на Firebase. Полный перенос функционала, интерфейса,
стилей и логики оригинала на React (функциональные компоненты + хуки).

## Запуск

```bash
npm install
npm run dev      # разработка (http://localhost:5173)
npm run build    # прод-сборка в dist/
npm run preview  # предпросмотр прод-сборки
```

Используется тот же Firebase-проект (`meelleniumsoft`), поэтому аккаунты, сайты, картинки и заявки —
общие с оригиналом.

## Маршруты

| Путь | Назначение |
|------|------------|
| `/` | приложение: вход (Google) → «Мои сайты» → редактор |
| `/view?id=<id>` | публичный просмотр опубликованного сайта |
| `/s/<slug>` | публичный просмотр по красивому адресу |
| `/example`, `/example-reg` | сайты-образцы |

## Структура

| Папка / файл | Назначение |
|--------------|------------|
| `src/engine/` | движок рендера сайта (`render.js`, `siteCss.js`, `defaultData.js`) — единый источник HTML для холста, предпросмотра, экспорта и публичного сайта. Порт `render.js` / `site-css.js` / `default-data.js` как ES-модули. |
| `src/cloud/` | Firebase: вход, Firestore (сайты, картинки, заявки, адреса), хук `useAuth`. Порт `cloud.js` на npm-пакет `firebase`. |
| `src/dialogs/` | кастомные диалоги (`uiAlert/uiConfirm/uiPrompt/uiLink`) как React-провайдер. |
| `src/dashboard/` | экраны «Вход» и «Мои сайты». |
| `src/editor/` | редактор: `Editor` (склейка + контекст), `Header`, `PagesBar`, `Stage` (холст-iframe со всеми взаимодействиями), `SidePanel` + `panels/` (настройки блоков, темы, стилей), модалки каталога/заявок/адреса, хранилище состояния `useSiteStore` (история Undo/Redo, автосохранение). |
| `src/view/` | публичный просмотр сайта и образцов. |

## Как устроен перенос

- **Движок рендера** оставлен покодовым портом оригинала: он генерирует автономный HTML сайта
  (в том числе для «Скачать ⤓» и опубликованного сайта), поэтому переписывать его в JSX было бы
  и вредно (сломало бы экспорт), и не нужно. Он подключён как ES-модули.
- **Холст редактора** — `<iframe srcdoc>` с тем же CSS сайта + служебные стили редактора; React-компонент
  `Stage` управляет пересборкой и навешивает обработчики (contenteditable, перетаскивание блоков/слоёв,
  замена фото) — поведение 1:1 с оригиналом.
- **Интерфейс** (шапка, панели настроек всех ~21 типов блоков, тема, модалки, диалоги, дашборд, вход)
  переписан на функциональные компоненты с переиспользуемыми полями (`fields.jsx`).

## Публикация на Firebase

```bash
npm run build
firebase deploy
```

`firebase.json` настроен как SPA (все пути → `/index.html`), хостинг раздаёт папку `dist/`.
